# 01 — 产品愿景与定位

## 1. 定位转变

| 维度 | 现在（v0.2） | 目标（v1.0 通用产品） |
|------|-------------|----------------------|
| 定位 | 作者自用开发效率工具 | 面向大众的轻量桌面任务与日历助理 |
| 用户 | 重度开发者 / 技术背景 | 知识工作者、学生、自由职业者、小团队个人用户 |
| 平台 | 仅 Windows 验证 | Windows 主力 + macOS 正式支持；Linux 社区版 |
| 数据 | 纯本地 SQLite | 本地为主，可选端到端加密同步 |
| 语言 | 界面繁简混用、写死文案 | 完整 i18n（简中 / 繁中 / 英文） |
| 分发 | 本地 MSI / NSIS 手动安装 | GitHub Releases + 官网下载 +（可选）应用商店 |
| 支持 | 无 | 用户文档、FAQ、Issue 模板、可选反馈通道 |
| 授权 | 个人项目、无 License | 明确开源或商业许可（见 §4） |

**一句话（v1.0）**：一款常驻桌面、在正确时间提醒你的**轻量任务与日历工具**——本地优先、跨平台、对非技术用户友好，而不是复杂的项目管理套件。

## 2. 目标用户画像

### 2.1 核心用户（Primary）

- **个人效率型**：需要「今天 / 本周 / 日历」一眼看清待办，依赖系统通知准时提醒。
- **特征**：任务量中等（几十～几百条），重视隐私与离线可用，不愿为简单 todo 付订阅或上云。
- **痛点**：Notion / Todoist 过重或需账号；系统自带提醒分散；日历与任务分离。

### 2.2 扩展用户（Secondary）

- **跨设备用户**：桌面主力 + 手机查看（v1.0 后通过同步或移动伴侣）。
- **中文语境用户**：农历、节假日、繁简区域设置（现有差异化优势）。
- **轻度团队**：共享只读清单或家庭待办（v2.0 愿景，非 v1.0 必做）。

### 2.3 非目标用户（明确不做）

- 需要甘特图、资源排期、依赖链的项目经理 → 用 Jira / Linear / MS Project。
- 需要实时多人协作编辑的企业团队 → 非 v1.0 范围。
- 需要重度统计分析、OKR 看板的管理者 → 保持「可操作工具」定位，不做统计图表堆砌。

## 3. 竞品与差异化

| 产品 | 优势 | 我们的差异 |
|------|------|-----------|
| Microsoft To Do | 免费、系统集成 | 更轻、日历月视图 + 农历、本地 DB 可备份、无微软账号 |
| Todoist / TickTick | 功能全、跨平台 | 本地优先、无强制云、桌面常驻轻量、开源/可自托管潜力 |
| Fantastical | 日历体验极佳 | 聚焦任务 + 提醒，体积与复杂度远低，Windows 原生 |
| Notion | 全能工作区 | 专注 todo + 提醒，启动快、心智负担低 |

**核心卖点（对外宣传用）**：

1. **轻**：~10MB 安装包，内存占用低（Tauri）。
2. **准**：系统原生通知 + 多提醒 + 休眠唤醒补发（Phase 1 补齐）。
3. **稳**：SQLite 本地库、自动备份、可导出（Phase 1–2）。
4. **懂中文**：农历、节假日、区域与繁简（持续打磨）。
5. **你的数据**：默认不上传；同步可选且端到端加密（Phase 3）。

## 4. 商业模式（待决策）

v1.0 前需拍板，影响功能拆分与品牌策略：

| 方案 | 说明 | 适用 |
|------|------|------|
| **A. 开源 + 捐赠** | MIT / Apache-2.0，GitHub Sponsors | 社区增长、个人品牌 |
| **B. 免费核心 + Pro** | 本地功能免费；同步、主题包、AI、协作 Pro 订阅 | 可持续收入 |
| **C. 一次性买断** | 各平台商店一次性付费 | 简单、无后端 |
| **D. 双轨** | 社区版开源 + 商店签名/自动更新/官方同步服务收费 | 常见 indie 路径 |

**建议**：Phase 2 结束前定案；开发上先按「功能模块可插拔」设计（同步、Pro 模块用 feature flag / 插件边界），避免返工。

## 5. 成功指标（v1.0）

| 类别 | 指标 | 目标 |
|------|------|------|
| 产品 | 核心流程完成率 | 新用户 10 分钟内完成：创建任务 → 设提醒 → 收到通知 |
| 质量 | 崩溃率 |  opt-in 遥测下 < 0.1% 会话崩溃（若启用 Sentry） |
| 质量 | 关键路径测试覆盖 | datetime / scheduler / migration 有自动化测试 |
| 平台 | Windows | Win10 21H2+、Win11 全功能验证 |
| 平台 | macOS | 通知、托盘、自启、打包 DMG 可用 |
| 工程 | 发布周期 | tag 推送后 30 分钟内 CI 产出安装包 |
| 增长 | 分发 | GitHub Releases + 官网 + README 安装说明完整 |
| 文档 | 用户向 | 快速开始、备份还原、FAQ、隐私说明 |

## 6. 品牌与包标识（Phase 2 前调整）

当前包名 `com.lenvovo.todo-reminder` 偏个人项目。通用产品化建议：

- [ ] 确定产品对外名称（Todo Reminder 或新品牌名）
- [ ] 统一 Bundle ID（如 `app.todoreminder.desktop`）
- [ ] 应用图标、托盘图标、通知 AUMID 一套视觉规范
- [ ] 官网域名 / GitHub Pages 落地页（可 v1.0 前极简单页）

> ⚠️ Bundle ID 变更会影响已有用户数据路径，需提供**一次性迁移**或文档说明。
