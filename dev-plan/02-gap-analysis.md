## 2. 功能缺口（产品化必补）

### 2.1 高优先级 — 用户会认为「半成品」

| 缺口 | 现状 | 产品化要求 |
|------|------|-----------|
| 子任务 UI | API 有，表单无 | 编辑任务时可增删改子项，卡片显示进度 |
| 标签 UI | API 有，筛选弱 | 多选标签、彩色 chip、设置页管理标签 |
| 拖拽 | 无 | 看板跨列、月历改期 |
| 多提醒 | 仅单条 upsert | 截止前 1 天 + 1 小时等多条 |
| 导入导出 | 无 | JSON 全量备份；Markdown 只读导出 |
| 数据安全 | 无自动备份 | 每日本地备份，保留 N 天 |
| 通知动作 | 计划有，未实现 | 通知上「完成 / 稍后 / 打开」 |
| 周期任务 | 无 | 日 / 周 / 月重复 |
| 全局热键 | 仅应用内 | 系统级快速新增（Ctrl+Shift+T） |

### 2.2 中优先级 — 通用产品体验

| 缺口 | 说明 |
|------|------|
| Undo 删除 | Toast 5 秒内可恢复 |
| 批量操作 | 多选完成 / 删除 / 改期 |
| 右键菜单 | 任务卡片快捷操作 |
| 设置项不足 | 时区、周起始日、默认提醒偏移、城市覆盖、通知音 |
| 活动日志 UI | 表已有，无「最近动态」面板 |
| 日详情面板 | 月历点击日期展开当日全部事件 |
| 错误文案 | 避免直接展示 SQL 错误 |

### 2.3 低优先级 — 差异化增强

| 缺口 | 说明 |
|------|------|
| 跨日任务连续色条 | 月历视觉，实现复杂 |
| 周 / 日视图 | Fantastical 级，todo 场景可延后 |
| Pomodoro | plan v0.3 提过 |
| 任务模板 | 常用任务一键创建 |

## 3. 体验与设计缺口

| 类别 | 问题 | 目标 |
|------|------|------|
| 设计系统 | 三套优先级色、shadow 不统一 | `design-tokens` 集中管理 |
| 新用户引导 | 有 onboarding 雏形 | 非技术用户 3 步上手 |
| 无障碍 | 月历无键盘导航 | Tab + 方向键；forced-colors 保底 |
| 触控 | 点击目标偏小 | 最小 44px 触控热区（可选） |
| Dark mode | Sidebar 与主区层次弱 | 主题感知 sidebar |
| i18n | 写死文案 | react-i18next，简繁英 |
| 帮助 | 无用户文档站 | 内置「帮助」或链到 docs |

## 4. 工程与质量缺口

| 类别 | 现状 | 产品化要求 |
|------|------|-----------|
| 单元测试 | 零 | vitest + cargo test 覆盖核心 lib / scheduler |
| CI/CD | 已推 GitHub，无 Actions | PR：lint + test；tag：构建安装包 |
| 跨平台构建 | 仅 Windows | macOS DMG；Linux AppImage/deb 可选 |
| 日志 | DEBUG 过多 | Release INFO；设置内「打开日志目录」 |
| 崩溃收集 | 无 | 可选 Sentry（默认关闭，隐私政策说明） |
| 性能 | 全量 loadAll | 月历按窗口查询；1k+ 任务压测 |
| 依赖更新 | 手动 | Dependabot |
| Win10 | 主要测 Win11 | 正式 Win10 矩阵测试 |

## 5. 商业化与合规缺口

| 类别 | 现状 | v1.0 前需要 |
|------|------|------------|
| LICENSE | README 写「暂无授权」 | 明确 MIT / 商业 / 双许可 |
| 隐私政策 | 无 | 说明本地存储、天气 IP 定位、可选遥测 |
| 第三方服务 | Open-Meteo、ipapi | 文档披露 + 失败降级 |
| 用户协议 | 无 | 若商店分发则需要 |
| 反馈渠道 | 无 | GitHub Issues 模板 + 可选邮件 |
| 更新机制 | 手动下载 | 应用内检查更新（Phase 3） |
| 代码签名 | 未签名 | Windows Authenticode、macOS 公证（减少 SmartScreen 警告） |

## 6. 架构预留（避免 v1.0 后大改）

产品化过程中建议提前抽象的边界：

```
┌─────────────────────────────────────────┐
│  UI (React) + i18n                       │
├─────────────────────────────────────────┤
│  Domain Services (TS stores + api/)      │
├─────────────────────────────────────────┤
│  Sync Adapter (interface, Phase 3 实现)   │  ← 新增
├─────────────────────────────────────────┤
│  Tauri IPC / Rust commands               │
├─────────────────────────────────────────┤
│  SQLite + File export/import             │
└─────────────────────────────────────────┘
```

- **Sync Adapter**：先定义 `pull/push/conflict` 接口，本地模式 no-op。
- **Settings 分层**：用户设置 vs 高级/实验功能 vs Pro 功能开关。
- **插件化提醒**：为后续 Webhook / 移动端推送留扩展点。
