## 阶段总览

```
v0.2.0 (现在)          自用 MVP，Windows only
    │
    ▼ Phase 1 · v0.3–v0.4   「好用」— 补齐核心功能与体验
    │
    ▼ Phase 2 · v0.5–v0.6   「可发布」— 多语言、CI、文档、macOS
    │
    ▼ Phase 3 · v0.7–v0.9   「可增长」— 同步、集成、性能、商店分发
    │
    ▼ Phase 4 · v1.0        「通用产品」— 跨平台稳定版 + 商业模式落地
    │
    ▼ Phase 5 · v2.0+       协作、AI、移动端（愿景）
```

## 原则

1. **本地优先**：默认零云端、数据在用户设备；同步与协作均为可选能力。
2. **轻量不打折**：保持 Tauri 体积与内存优势，不为了功能堆成 Electron 体量。
3. **先 Windows 再跨平台**：Windows 体验打磨到位后再签 macOS / Linux，避免三端同时半成品。
4. **小步发布**：每个 minor 版本有明确主题、可独立安装验证，CHANGELOG 对用户可读。
5. **工程债与功能并行**：每个阶段预留 20–30% 工时给测试、CI、设计系统，不单堆功能。

## 建议起步

若只选一件事开始 Phase 1，按 ROI 排序：

1. 子任务 UI + 标签 UI（schema 已备，用户感知最强）
2. 看板 / 月历拖拽改期
3. 自动 DB 备份 + 导入导出
4. 设计 token 统一 + 错误文案友好化

详细任务拆解与工时见 [03-roadmap.md](./03-roadmap.md)。
